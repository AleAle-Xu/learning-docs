# SSH 端口转发详解

SSH 端口转发（Port Forwarding）是 SSH 提供的隧道功能，可以将网络流量通过加密的 SSH 连接进行中转，用于访问内网服务、穿透防火墙或实现流量代理。

共有三种模式：`-L`（本地转发）、`-R`（远程转发）、`-D`（动态转发）。

---

## 通用参数

端口转发时常配合以下参数使用：

| 参数 | 说明 |
|------|------|
| `-N` | 不执行远程命令，仅建立隧道（转发专用时推荐加上） |
| `-f` | 后台运行 |
| `-C` | 压缩传输数据 |
| `-o ServerAliveInterval=60` | 每 60 秒发送心跳，防止连接超时断开 |

典型组合：

```bash
ssh -NfC -L ... user@jump-host
```

## -L 本地端口转发

### 原理

将**本机**的某个端口，通过 SSH 服务器，转发到**目标主机**的某个端口。

```
本机:本地端口 ──SSH隧道──▶ SSH服务器 ──▶ 目标主机:目标端口
```

访问"本机本地端口"等同于从 SSH 服务器访问"目标主机:目标端口"。

### 语法

```bash
ssh -L [本机绑定地址:]本地端口:目标主机:目标端口 user@SSH服务器
```

> `本机绑定地址` 默认为 `127.0.0.1`（只有本机能访问），设为 `0.0.0.0` 则局域网内其他机器也能访问。

### 典型场景

**场景 1：访问远程服务器上的服务（最常见，两机直连）**

远程服务器 `remoteA` 上跑了一个服务监听在 `8000` 端口，但该端口没有对外暴露，只能在服务器本机访问。

```bash
ssh -NL 8000:localhost:8000 user@remoteA
```

之后在本地浏览器访问 `http://localhost:8000` 即可。

```
本机:8000 ──SSH隧道──▶ remoteA ──▶ remoteA的localhost:8000
                           └─ 目标就是它自己，无需第三台机器
```

> 这里 `localhost` 是**相对于 remoteA** 而言的，即 remoteA 自己的 localhost。这是最高频的两机直连用法，SSH 服务器本身就是服务所在的目标机器。

**场景 2：通过跳板机访问内网服务（三机）**

公司内网有一台 MySQL（`192.168.1.100:3306`），只能从跳板机（`jump.example.com`）访问，本地无法直连。

```bash
ssh -NL 3307:192.168.1.100:3306 user@jump.example.com
```

之后在本地用 `mysql -h 127.0.0.1 -P 3307` 即可连接内网数据库。

```
本机:3307 ──SSH──▶ jump.example.com ──▶ 192.168.1.100:3306
                        └─ 跳板机转发到第三台机器
```

**场景 3：访问远程服务器上非 localhost 绑定的服务**

如果目标服务绑定在远程服务器的某个具体 IP 或其他接口上，`localhost` 换成对应地址即可，用法与上面一致。

## -R 远程端口转发

### 原理

将**SSH 服务器**的某个端口，转发回**本机**的某个端口。方向与 `-L` 相反。

```
SSH服务器:远程端口 ──SSH隧道──▶ 本机:本地端口
```

常用于：**让外网服务器能访问本机服务**（内网穿透）。

### 语法

```bash
ssh -R [远程绑定地址:]远程端口:目标主机:目标端口 user@SSH服务器
```

> 若希望 SSH 服务器上除了 `localhost` 之外的地址也能访问该端口，需在服务端 `/etc/ssh/sshd_config` 中开启 `GatewayPorts yes`。

### 典型场景

**场景：内网穿透，让外网访问本机服务**

本机开发了一个服务跑在 `localhost:3000`，想让有公网 IP 的服务器（`vps.example.com`）对外暴露：

```bash
ssh -NR 9000:localhost:3000 user@vps.example.com
```

之后访问 `http://vps.example.com:9000` 就会被转发到本机的 3000 端口。

```
外网用户 ──▶ vps.example.com:9000 ──SSH隧道──▶ 本机:3000
```

## -D 动态端口转发（SOCKS 代理）

### 原理

在本机启动一个 **SOCKS5 代理服务器**，所有发往该端口的流量，都通过 SSH 隧道由远程服务器转发出去。

```
本机应用 ──SOCKS5──▶ 本机:本地端口 ──SSH隧道──▶ SSH服务器 ──▶ 目标网络
```

与 `-L` 的区别：`-L` 只能转发到固定的一个目标；`-D` 是动态的，能代理任意目标地址（由 SOCKS 协议决定路由）。

### 语法

```bash
ssh -ND 本地端口 user@SSH服务器
```

### 典型场景

**场景：通过服务器访问其所在网络内的任意资源**

```bash
ssh -ND 1080 user@jump.example.com
```

然后将本机浏览器或应用的代理设置为 `SOCKS5 127.0.0.1:1080`，所有流量将经由 `jump.example.com` 转发，访问该服务器能访问的任意地址。

---

## 三种模式对比

| 模式 | 转发方向 | 适用场景 |
|------|---------|---------|
| `-L 本地端口:目标:端口` | 本机 → 远端 | 访问内网服务、访问远程 localhost 服务 |
| `-R 远程端口:目标:端口` | 远端 → 本机 | 内网穿透，让外网访问本机服务 |
| `-D 本地端口` | 本机 → 任意（动态） | SOCKS 代理，代理整个网络流量 |

## 持久化配置（~/.ssh/config）

每次都手敲参数很麻烦，常用的转发可以写入 `~/.ssh/config`：

```
Host db-tunnel
    HostName jump.example.com
    User user
    LocalForward 3307 192.168.1.100:3306
    ServerAliveInterval 60
    ServerAliveCountMax 3
```

之后直接执行 `ssh -N db-tunnel` 即可建立隧道。

## 常见问题

**端口被占用**

```bash
# 查看哪个进程占用了端口
ss -tlnp | grep :3307

# 找到 PID 后杀掉
kill <PID>
```

**连接容易超时断开**

加上心跳参数或写入 config：

```bash
ssh -o ServerAliveInterval=60 -o ServerAliveCountMax=3 -NL ...
```

**`-R` 转发后外网访问不了**

检查 SSH 服务器的 `/etc/ssh/sshd_config`，确保有：

```
GatewayPorts yes
```

修改后重启 sshd：`sudo systemctl restart sshd`
